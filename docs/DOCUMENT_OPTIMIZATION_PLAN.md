# 문서 최적화 계획 (토큰 수 감소)

## 목표
Agent에서 사용되는 토큰 수를 줄이기 위해 문서를 간결하고 핵심적인 내용만 유지하도록 개선

## 개선 원칙
1. **중복 제거**: 동일한 내용이 여러 문서에 반복되는 경우 통합
2. **완료된 작업 축소**: 완료된 Phase의 상세 작업 내역은 요약만 유지
3. **예제 간소화**: 긴 예제 코드는 핵심만 유지
4. **핵심 정보 우선**: 필수 정보만 유지, 부가 설명 축소
5. **참조 활용**: 상세 내용은 다른 문서 참조로 대체

## 개선 대상 문서 및 계획

### 1. TODO.md (현재: 631줄 → 목표: ~300줄, 52% 감소)

#### 개선 사항
- ✅ **완료된 Phase 요약 섹션 유지** (현재 상태 유지)
- ❌ **완료된 Phase의 상세 Step 내역 제거**
  - Phase 4의 Step 1-8 상세 작업 내역 삭제
  - Acceptance Criteria만 요약으로 유지
- ✅ **진행 중/예정 Phase는 상세 유지**
- ❌ **중복된 "최종 목표" 섹션 축소**

#### 변경 내용
```diff
- Phase 4 Step 1-8의 상세 작업 내역 (약 200줄)
+ Phase 4 완료 요약만 유지 (약 20줄)
- "최종 목표" 섹션 중복 내용 제거
+ 핵심 목표만 유지
```

### 2. PROGRESS.md (현재: 542줄 → 목표: ~200줄, 63% 감소)

#### 개선 사항
- ✅ **핵심 달성 사항만 유지**
  - 테스트 통과 수, 커버리지, API 엔드포인트 수
- ❌ **상세 구현 내용 제거**
  - "주요 구현 내용" 섹션 축소
  - "서브 프로젝트" 상세 설명 제거
- ✅ **날짜별 요약만 유지**
- ❌ **성능 지표는 핵심 수치만**

#### 변경 내용
```diff
- 각 Phase의 "주요 구현 내용" 상세 설명
+ 핵심 달성 사항만 표 형식으로 정리
- "서브 프로젝트" 섹션 제거
+ 필요시 TODO.md 참조로 대체
```

### 3. DEFINITION_OF_DONE.md (현재: 359줄 → 목표: ~200줄, 44% 감소)

#### 개선 사항

- ❌ **반복되는 설명 제거**
  - "테스트 우선" 원칙이 여러 곳에 반복
- ❌ **예제 코드 간소화**
  - 긴 코드 블록은 핵심만 유지
- ✅ **Phase별 DoD는 핵심 기준만**
- ❌ **체크리스트 사용 방법 축소**

#### 변경 내용
```diff

+ 핵심 원칙만 명시
- 긴 코드 예제 (bash 명령어 등)
+ 핵심 명령어만 유지
```

### 4. DEVELOPMENT_GUIDE.md (현재: 349줄 → 목표: ~250줄, 28% 감소)

#### 개선 사항
- ✅ **코딩 컨벤션 유지**
- ❌ **예제 코드 간소화**
  - 함수/컴포넌트 예제는 핵심 패턴만
- ❌ **"실제 구현 예시" 참조 제거**
  - 코드베이스 참조 문구 반복 제거
- ✅ **네이밍 규칙, Git 워크플로우 유지**
- ❌ **테스트 전략 섹션 축소**
  - DEFINITION_OF_DONE.md와 중복 내용 제거

#### 변경 내용
```diff
- 긴 예제 코드 블록 (20줄 이상)
+ 핵심 패턴만 5-10줄로 축소
- "실제 구현 예시: backend/app/..." 반복 참조
+ 한 번만 명시하고 나머지 제거
- 테스트 전략 상세 설명
+ DEFINITION_OF_DONE.md 참조로 대체
```

## 개선 시 주의사항

1. **핵심 정보 보존**: 필수 정보는 반드시 유지
2. **참조 링크 유지**: 다른 문서 참조 링크는 그대로 유지
3. **완료 기준 명확**: Definition of Done의 핵심 기준은 유지
4. **코드 예제 최소화**: 핵심 패턴만 보여주고 나머지는 코드베이스 참조
5. **일관성 유지**: 축소 후에도 문서 간 일관성 유지
