# IntradayChart: 분봉 차트

> 📄 **상위 문서**: [ETF_DETAIL.md](./2.ETF_DETAIL.md) § 3-7

당일 **시간별 체결 데이터**를 시각화하는 분봉 차트. 장중/장외에 따라 자동 갱신·staleTime이 다르며, 전일 종가·피봇 레벨·목표가 수평선을 표시한다.

---

## 1. 조회 방식 개요

- **API**: `GET /api/etfs/{ticker}/intraday`
- **파라미터**: `target_date`(선택), `auto_collect`(기본 true), `force_refresh`(캐시 무시)
- **enabled**: `!!ticker && !!etf` — 종목 기본 정보 로드 후에만 요청

### 프론트엔드 (ETFDetail.jsx)

| 조건 | refetchInterval | staleTime |
|------|-----------------|-----------|
| 장중 (09:00~15:30, 평일) | 20초 | 10초 |
| 수집 중 (background_collect_started) | 3초 | — |
| 장외 | false (끔) | 30초 |

- `refetchOnMount: true`
- **수동 새로고침**: `forceRefreshRef.current = true` 후 refetchIntraday() → `force_refresh=true` 전송

## 2. 표시 요소

| 요소 | 소스 | 조건 |
|------|------|------|
| 전일 종가 수평선 | previousClose (pricesData[1]?.close_price) | previousClose 있을 때 |
| 피봇 레벨 | supportResistanceData?.pivot | showAdvanced(고급분석) 시 |
| 목표가 수평선 | alertRules (PriceTargetPanel과 동일) | buy/sell 타입, is_active |
| 차트 데이터 | intradayData.data | 시가·고가·저가·종가·거래량·시간 |

---

## 3. IntradayChart 컴포넌트 Props

| Prop | 타입 | 용도 |
|------|------|------|
| data | array | 분봉 배열 [{ time, open, high, low, close, volume, price }] |
| ticker | string | 종목 코드 |
| height | number | 차트 높이 (기본 300) |
| showVolume | boolean | 거래량 표시 여부 |
| previousClose | number | 전일 종가 수평선 |
| pivotLevels | object | 피봇 R1~R3, P, S1~S3 |
| priceTargets | array | 알림 규칙 (목표가) |

## 5. 장중 여부 판단

```javascript
// 09:00 = 540분, 15:30 = 930분
const isMarketHours = (날짜 1~5) && (시간 540~930분)
// 1분마다 setInterval로 재확인
```

---

## 6. 백엔드 조회·수집 흐름 (요약)

1. 캐시 확인 (장중 15초 / 장외 30초 TTL). force_refresh 시 캐시 무효화
2. DB 조회 (09:00~15:30 범위)
3. 데이터 없으면 마지막 거래일 재조회
4. 장중 재수집: 마지막 체결 3분 경과 시 재수집 트리거
5. auto_collect=true이고 데이터 없거나 재수집 필요 시 → 백그라운드 수집 (전체 40페이지 또는 증분 최대 10페이지)
6. 빈 결과는 **캐시하지 않음** (다음 요청에서 자동 수집 재시도)


---

## 7. 문제 해결

| 현상 | 원인 | 조치 |
|------|------|------|
| 장중 자동 갱신 안 됨 | isMarketHours 오류 또는 refetchInterval 미적용 | 1분마다 재확인 로직, useQuery 옵션 확인 |
| 분봉 수집 안 됨 | 빈 결과 캐시 | 백엔드에서 빈 결과 미캐시 (이미 적용) |
| 수집 중단 | 타임아웃 | intraday API 타임아웃 확대 (LONG_API_TIMEOUT) |
| 장 시작 데이터 누락 | 페이지 수 부족 | 백엔드 40페이지+α 자동 수집 |
| 목표가 선 안 보임 | alertRules 미전달 | useQuery ['alertRules', ticker] 결과 전달 확인 |
