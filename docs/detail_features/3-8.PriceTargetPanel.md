# PriceTargetPanel: 알림 설정 패널

> 📄 **상위 문서**: [ETF_DETAIL.md](./2.ETF_DETAIL.md) § 3-8

분봉 차트 하단에 위치하는 **3탭 알림 설정 패널**. 목표가·급등/급락·매매시그널 규칙의 CRUD, 활성/비활성 토글, 현재가 기준 괴리율 표시. Optimistic Update로 즉시 UI 반영.

---

## 1. 동작 방식 개요

- **위치**: 분봉 차트 카드 내부, IntradayChart·목표가 설명 하단
- **API**: `GET /api/alerts/{ticker}` (규칙 조회), create/update/delete 엔드포인트
- **캐시**: queryKey `['alertRules', ticker]`, staleTime 30초
- **currentPrice**: 분봉 최신가 또는 pricesData[0].close_price (분봉 없을 때)

---

## 2. 탭별 알림 유형

| 탭 key | label | alert_type | direction | target_price |
|--------|-------|------------|-----------|--------------|
| target | 목표가 | buy / sell | below / above | 금액 |
| change | 급등/급락 | price_change | above / below / both | 임계 % |
| signal | 매매시그널 | trading_signal | above / below / both | 0 (미사용) |

---

## 3. CRUD 및 뮤테이션

| 동작 | API | onSuccess |
|------|-----|-----------|
| 생성 | alertApi.createRule(payload) | invalidate, resetForm |
| 수정 | alertApi.updateRule(id, data) | invalidate, setEditingId(null), resetForm |
| 삭제 | alertApi.deleteRule(id) | invalidate |
| 토글 | alertApi.updateRule(id, { is_active }) | invalidate |

---

## 4. 폼 기본값 (FORM_DEFAULTS)

```javascript
buy:             { alert_type: 'buy',             direction: 'below', target_price: '', memo: '' }
sell:            { alert_type: 'sell',            direction: 'above', target_price: '', memo: '' }
price_change:    { alert_type: 'price_change',    direction: 'both',  target_price: '', memo: '' }
trading_signal:  { alert_type: 'trading_signal',  direction: 'both',  target_price: '0', memo: '' }
```

---

## 5. 괴리율 계산

```javascript
calcDiffPct(targetPrice) = ((targetPrice - currentPrice) / currentPrice * 100).toFixed(1)
```

- 목표가 탭: 각 규칙별 현재가 대비 괴리율 % 표시
- 급등/급락: target_price가 임계 %이므로 괴리율 대신 임계값 표시

---

## 6. Submit 검증

- trading_signal: target_price 0 고정, 별도 검증 없음
- price_change: target_price는 parseFloat (임계 %)
- buy/sell: target_price는 fromCommaString (금액), NaN 또는 ≤0이면 제출 무시

---

## 7. Optimistic Update

- 뮤테이션 호출 후 서버 응답 전에 `invalidateQueries`로 쿼리 무효화 → refetch
- UI는 refetch 완료 시 갱신 (실제 Optimistic Update는 별도 구현 없이 invalidate로 처리)

---

## 8. 문제 해결

| 현상 | 원인 | 조치 |
|------|------|------|
| 규칙 목록 안 나옴 | alertRules 쿼리 에러 | Network 탭, 백엔드 로그 확인 |
| 괴리율 NaN | currentPrice null | 분봉 또는 일봉 종가 전달 확인 |
| 수정 시 폼 초기화 안 됨 | resetForm/ setEditingId 미호출 | onSuccess 핸들러 확인 |
| 토글 반영 안 됨 | invalidate 누락 | toggleMutation onSuccess에 invalidate |
