# DateRangeSelector: 날짜 범위 선택

> 📄 **상위 문서**: [ETF_DETAIL.md](./2.ETF_DETAIL.md) § 3-5

가격·매매동향 차트의 **분석 기간**을 선택하는 컴포넌트. 7일/1개월/3개월/6개월/YTD/1년 프리셋과 커스텀 날짜 입력을 제공하며, 설정 페이지의 기본값과 연동한다.

---

## 1. 동작 방식 개요

- **프리셋**: 7d, 1m, 3m, 6m, ytd, 1y, custom
- **연동**: `defaultRange` prop으로 설정값(7D→7d, 1M→1m, 3M→3m) 전달
- **콜백**: `onDateRangeChange({ startDate, endDate, range })` 호출
- **검증**: 커스텀 날짜는 `validateDateRange`로 검증, `MAX_DATE_RANGE_DAYS` 제한

### ETFDetail 연동

- `userModifiedRef`: 사용자가 수동 변경 시 true → 설정 기본값으로 덮어쓰지 않음
- `key`: `defaultRangeFromSettings` 변경 시 컴포넌트 리마운트하여 새 기본값 반영
- queryKey: `dateRange.startDate`, `dateRange.endDate` 포함 → 범위 변경 시 가격·매매동향 자동 재요청

---

## 2. 프리셋별 계산

| range | 계산 | date-fns |
|-------|------|----------|
| 7d | 오늘 - 7일 | subDays(today, 7) |
| 1m | 오늘 - 1개월 | subMonths(today, 1) |
| 3m | 오늘 - 3개월 | subMonths(today, 3) |
| 6m | 오늘 - 6개월 | subMonths(today, 6) |
| ytd | 올해 1월 1일 | startOfYear(today) |
| 1y | 오늘 - 1년 | subYears(today, 1) |
| custom | 사용자 입력 | — |

종료일은 항상 `format(today, 'yyyy-MM-dd')`.

---

## 3. 커스텀 날짜

- **입력**: startDate, endDate (type="date")
- **적용 버튼**: 클릭 시 `applyCustomRange()` 호출
- **검증**: `validateDateRange(startDate, endDate)` — isValid, error 반환
- **에러 표시**: `bg-red-50 border-red-200` 박스에 error 메시지
- **선택된 기간 표시**: custom이 아닐 때 "선택된 기간: {startDate} ~ {endDate}"

---

## 4. 초기화 및 defaultRange 반영

```javascript
useEffect(() => {
  if (initialStartDate && initialEndDate) return // 부모가 이미 설정한 경우 스킵
  if (defaultRange !== 'custom') {
    handlePresetClick(defaultRange)
  }
}, [defaultRange, initialStartDate, initialEndDate])
```

- 설정 페이지에서 defaultDateRange 변경 시 ETFDetail이 `key`를 바꿔 리마운트 → 새 defaultRange 적용
- `userModifiedRef`가 true면 ETFDetail의 useEffect에서 defaultRange 업데이트를 무시

---

## 5. 문제 해결

| 현상 | 원인 | 조치 |
|------|------|------|
| 설정 변경해도 범위 안 바뀜 | userModifiedRef가 이미 true | 초기화 버튼 추가 또는 페이지 새로고침 |
| 커스텀 적용 시 에러 | 날짜 역전, 범위 초과 | validateDateRange, MAX_DATE_RANGE_DAYS 확인 |
| 기간 표시가 잘못됨 | date-fns 로케일/타임존 | format(today, 'yyyy-MM-dd') 일관 적용 |
